4. The Three Synthesis Layers (LOCKED DEFINITIONS)
4.1 Emotional Inference — Posture-Only
What this is NOT

No diagnosing

No emotion labels

No claims like “you are anxious / overwhelmed”

What this IS

Detecting uncertainty or cognitive load via language signals

Adjusting tone, pacing, and offered options only

Posture-only definition:

The AI may change how it responds, but not what it concludes, recommends, or decides based on inferred emotion.

Examples

Offering to unpack vs answering directly

Suggesting holding something as a decision

Slower, gentler language when uncertainty is high

Why this is safe

No memory writes

No emotional claims

No loss of user agency

4.2 Relational Synthesis — Recognition-First
Critical clarification

Most relational data already exists.
The missing piece is how it is surfaced, not new data.

Allowed behaviour

“There are related decisions in Family and Money.”

“You’ve filed similar items under this area.”

This is recognition, not reasoning.

Hard constraints

No conclusions

No “therefore you should”

No inferred causality

No prioritisation

UI constraint (LOCKED)

Relational surfacing must be:

Sparse

Optional

Capped (e.g. max 1–2 gentle references per response)

Over-surfacing breaks calm.

4.3 Temporal Synthesis — Facts Only

This layer is powerful and intentionally limited in V1.

Allowed

“You last reviewed this on…”

“This has come up before.”

“You scheduled this for…”

Explicitly NOT allowed (V1)

Pattern judgement

Behavioural narratives

Identity claims (e.g. “you always delay”)

Reason for restriction

Temporal synthesis is facts-only to prevent premature identity or behavioural narratives from forming.
